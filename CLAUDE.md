## ü§ñ Stateless Agent Architecture - Core Principles

**Problem**: AI agents have no memory between sessions. Investment in "teaching" evaporates.

**Solution**: Design systems that work WITH statelessness, not against it.

### The Four Principles

1. **Interface Explicit** - Enforce through types/schemas/MCP, not conventions
2. **Context Embedded** - Travel with code (comments, config), not external docs
3. **Constraints Automated** - Hooks/validators prevent mistakes structurally
4. **Iteration Protocol** - Error ‚Üí System patch (not agent re-education)

### The Three-Layer Solution

**Hooks** (Automatic) - Same thing every time (validation, context injection)
**Skills** (Workflow) - Multi-step procedures (reusable patterns)
**MCP** (Data Access) - External data discovery (runtime, no hardcoding)

**Full Framework**: See `.hermetic/docs/stateless-architecture-mapping.md`
