## Learn Any Skill Fast - Hermetic Learning Workflow

**Domain:** Skill Acquisition & Learning
**Principles Applied:** Mentalism (mental models) + Vibration (rapid practice) + Rhythm (sustainable pace) + Gender (input/output balance)

### Purpose
Learn new skills efficiently by building clear mental models, practicing rapidly, honoring learning cycles, and balancing input with output.

### When to Use
- Learning new programming language/framework
- Acquiring new technical skill
- Mastering new domain knowledge
- Career development
- Personal growth

### The Hermetic Learning Process

**Step 1: MENTALISM - Build Mental Model First**
```
Before diving into practice, understand the STRUCTURE:

1. What is this skill fundamentally?
   Example: "React is a UI library using components and state"
   (Not: "React is complicated JavaScript framework thing")

2. What are the core concepts? (usually 5-7)
   React: Components, Props, State, Hooks, JSX, Events, Lifecycle

3. How do concepts relate?
   Components render JSX → Props pass data → State triggers re-render

4. Draw the mental model
5. Refine until crystal clear

Clear mental model = 10x faster learning
```

**Step 2: VIBRATION - Practice Immediately**
```
Don't study endlessly. Practice NOW:

1. Find SMALLEST practical project
   Example: "Todo list" not "Full social network"

2. Build it IMMEDIATELY (today)
3. Struggle is learning (embrace confusion)
4. Make mistakes fast (feedback loop)
5. Ship something, anything

Learning happens through DOING, not reading
```

**Step 3: RHYTHM - Sustainable Learning Pace**
```
Build learning into natural cycles:

DAILY:
- 90 min focused learning (deep work)
- 15 min integration (review what you learned)
- Repeat 1-2 cycles max
- NO marathon 8-hour sessions (burnout)

WEEKLY:
- 5 days active learning
- 2 days integration (let it sink in)
- Brain needs rest to consolidate

MONTHLY:
- Week 1-3: Intense practice
- Week 4: Light practice, mostly review
- Rhythmic intensity prevents burnout

Consistency > Intensity
1 hour daily for 30 days beats 30 hours in 1 weekend
```

**Step 4: GENDER - Balance Input & Output**
```
Alternate between receiving (feminine) and creating (masculine):

FEMININE (Input):
- Read documentation
- Watch tutorials
- Study examples
- Absorb concepts
- Listen to experts

MASCULINE (Output):
- Build projects
- Write code
- Create applications
- Teach others
- Produce work

BALANCE:
- 30% input (learning concepts)
- 70% output (applying immediately)

Too much input → Paralysis (tutorial hell)
Too much output → Shallow understanding
Balance both → Deep mastery
```

**Step 5: PROGRESSIVE COMPLEXITY**
```
Using Vibration (movement) to build momentum:

Level 1: Tutorial project (follow along)
  ↓ Ship it
Level 2: Modify tutorial (change it)
  ↓ Ship it
Level 3: Original simple project (apply concepts)
  ↓ Ship it
Level 4: Original complex project (stretch)
  ↓ Ship it
Level 5: Teach others (ultimate test)

Each level builds on previous
Don't skip levels (foundation matters)
```

**Step 6: DELIBERATE PRACTICE**
```
Using Mentalism (conscious focus):

1. Identify weakness in your understanding
2. Design exercise specifically targeting that weakness
3. Practice that exercise repeatedly
4. Get feedback (code review, mentor, tests)
5. Refine understanding based on feedback

Generic practice → Slow progress
Deliberate practice → Rapid improvement
```

**Step 7: TEACH TO MASTER**
```
Using Correspondence (as you teach, so you understand):

1. Explain concept to someone else (or rubber duck)
2. If you can't explain simply, you don't understand deeply
3. Write blog post/tutorial (output)
4. Answer questions in community (helps others, deepens yours)

Teaching is the ultimate test of understanding
```

### Expected Outcome
- Rapid skill acquisition (weeks not years)
- Deep understanding (not surface knowledge)
- Practical ability (can actually build things)
- Sustainable pace (no burnout)
- Long-term retention (understanding vs memorization)

### Example Application

**Goal:** Learn Python

**Week 1 - Mentalism (Build Mental Model):**
- Core concepts: Variables, Functions, Classes, Modules, Loops
- Mental model: "Python is interpreted scripting language with dynamic typing"
- 2 days reading docs, understanding structure

**Week 1-2 - Vibration (Rapid Practice):**
- Day 3: Build calculator CLI
- Day 4: Build todo list app
- Day 5: Build simple web scraper
- Day 6-7: Integration, review
→ 4 projects in 5 days

**Week 2-4 - Rhythm (Sustainable Pace):**
- Daily: 90 min focused practice
- Build 1 small project every 2 days
- Review on weekends
→ 12 projects in 3 weeks

**Throughout - Gender (Input/Output Balance):**
- 30% reading docs/tutorials (input)
- 70% building projects (output)
→ Deep practical understanding

**Week 4 - Teach (Master It):**
- Write tutorial on Python basics
- Help friend learn Python
- Answer questions on Stack Overflow
→ Mastery through teaching

**Result:** Functional Python developer in 4 weeks

### Hermetic Truth
> "Learn the structure (Mentalism), practice immediately (Vibration), pace sustainably (Rhythm), and balance input with output (Gender). Mastery follows."
