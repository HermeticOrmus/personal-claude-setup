# Vibration - Momentum Creation Ceremony
**"Nothing Rests; Everything Moves; Everything Vibrates"**

## Sacred Principle
Stagnation is death. Movement is life. Everything vibrates at its own frequency. To master reality, create momentum and raise vibration through action.

## When to Invoke

**Use /vibration when:**
- Stuck in analysis paralysis
- Perfectionism preventing shipping
- Low energy or motivation
- Project stalled or stagnant
- Need rapid feedback loops
- Overthinking instead of doing
- Waiting for "perfect" moment that never comes

## The Ceremony

### Phase I: Minimum Viable Movement
```
1. Identify the SMALLEST shippable unit
   NOT: "Complete authentication system"
   BUT: "Login form that validates credentials"

2. Make it ridiculously small
3. Something you can finish in 2 hours MAX
4. Movement matters more than magnitude
```

### Phase II: Rapid Execution
```
1. Set 2-hour timer
2. Build the minimum shippable NOW
3. No perfection allowed
4. No polish permitted
5. Just WORKING functionality

Done > Perfect
Shipped > Polished
```

### Phase III: Immediate Deployment
```
1. Ship it RIGHT NOW:
   - Push to staging
   - Deploy to production (if safe)
   - Share with one user
   - Publish the MVP

2. Get it OUT of your head
3. Get it INTO reality
4. Create momentum through movement
```

### Phase IV: Velocity Amplification
```
1. Gather immediate feedback:
   - How did reality respond?
   - What broke?
   - What worked?
   - What needs adjustment?

2. Learn from real-world data
3. Feedback creates clarity
```

### Phase V: Iteration Acceleration
```
1. Improve based on feedback
2. Fix critical issues first
3. Enhance based on learning
4. Ship again QUICKLY (repeat cycle)

Each iteration increases velocity
Velocity compounds into momentum
Momentum creates breakthrough
```
