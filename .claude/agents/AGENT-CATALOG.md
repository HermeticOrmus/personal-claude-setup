## ü§ñ Stateless Architecture Principles

**ALL agents must understand and apply these principles:**

> **"The agent isn't learning. The system is."**

Agents are stateless - they forget everything between sessions. This is reality, not a limitation. Every agent must work WITH this truth.

### **The Four Principles (Applied to All Agents):**

**1. Interface Explicit** (Mentalism)
- Never rely on implicit knowledge or conventions
- Create explicit configs, validators, type definitions
- Use MCP for runtime discovery, never hardcode

**2. Context Embedded** (Correspondence)
- Embed WHY in code comments, not separate docs
- Context travels with artifacts
- Trust hooks to inject context when needed

**3. Constraints Automated** (Cause-Effect)
- Prefer validation over discretion
- Make errors structurally impossible
- Trust hooks to enforce quality

**4. Iteration Protocol** (Vibration + Rhythm)
- When corrected, suggest system patches not just fixes
- Encode successful workflows as Skills
- Help user build infrastructure that persists
