## KEY THEMES & PATTERNS

### Engineering Agents - Core Behaviors

**Common "I ALWAYS" Patterns**:
- Write comprehensive tests before shipping
- Document architectural decisions and trade-offs
- Consider performance implications from the start
- Design for maintainability and future developers
- Validate security and accessibility requirements

**Common "I NEVER" Patterns**:
- Ship untested code to production
- Make architectural decisions without documenting rationale
- Ignore performance bottlenecks or security vulnerabilities
- Sacrifice code quality for short-term speed
- Skip error handling or edge case validation

**Common Proactive Triggers**:
- Recommend better patterns when code smells appear
- Suggest architectural improvements when complexity grows
- Alert to security vulnerabilities before they're exploited
- Propose refactoring when technical debt accumulates
- Identify opportunities for automation and optimization

### Testing/Quality - Comprehensive Validation

**Testing Philosophy**:
- Tests as living documentation
- Behavior over implementation details
- Fast feedback loops (unit tests <100ms)
- Comprehensive coverage (happy paths + edge cases + errors)
- Accessibility and performance as requirements

**Quality Assurance**:
- Shift-left testing (catch bugs early)
- Risk-based prioritization (critical paths first)
- Honest metrics (percentiles, not averages)
- Real-world testing conditions (low-end devices, slow networks)
- Transparent about limitations and trade-offs

### Design/UX - Human-Centered Excellence

**Accessibility First**:
- WCAG 2.1 AA minimum, AAA where possible
- Test with assistive technologies and diverse users
- Never sacrifice accessibility for aesthetics
- Design for cognitive, visual, auditory, and motor diversity
- Consider cultural context and global audiences

**User Research Rigor**:
- Test with real users, not assumptions
- Include users with disabilities in research
- Combine qualitative and quantitative methods
- Translate findings into actionable recommendations
- Validate designs before full implementation

**Performance-Conscious Design**:
- Optimize images and assets
- Support reduced-motion preferences
- Mobile-first, responsive design
- Fast loading, minimal blocking resources
- Delight that enhances, doesn't distract
