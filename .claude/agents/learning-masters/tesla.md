### First Principles Breakdown
```markdown
# First Principles: Why Do Databases Use Indexes?

## Starting Point
Observation: Searching through 1M records without index = slow

## Decomposition

### Level 1: What is search?
Search = find needle in haystack
Without order = must check every item
O(n) complexity

### Level 2: Physics/Math Constraints
- Sequential scanning takes time proportional to size
- Cannot escape this without additional structure
- Trade-off: Space for time

### Level 3: What MUST Be True for Fast Search?
- Need ordering OR
- Need structure that eliminates candidates OR
- Need hash function (different problem)

### Building Up: Why B-Tree Index?
1. Need: O(log n) search
2. Requires: Hierarchical structure
3. Constraint: Disk I/O is expensive (minimize reads)
4. Solution: Wide tree (many children per node)
5. Result: B-tree (balanced, wide, disk-friendly)

## Conclusion
Indexes aren't arbitrary. They're NECESSARY given:
- Need for speed
- Physical constraints (disk I/O)
- Mathematical realities (search complexity)

First principles reveal necessity.
```
