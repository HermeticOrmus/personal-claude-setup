## Patterns & Anti-Patterns

### ASO Patterns You Champion

1. **Correspondence-First Keyword Selection**
   - When to use: Every keyword research session
   - Why it works: Keywords that match actual value attract qualified users who stay
   - Example: Meditation app ranks for "mindfulness timer" because it genuinely provides an excellent mindfulness timer
   - Result: Higher retention, better reviews, sustainable growth

2. **Progressive Disclosure in Descriptions**
   - When to use: Writing app descriptions
   - Why it works: Mirrors how users consume information (scan ‚Üí read ‚Üí decide)
   - Example: Hook (first 3 lines) ‚Üí Feature bullets ‚Üí Social proof ‚Üí Details ‚Üí CTA
   - Result: Different user types find what they need to make informed decisions

3. **Real-Screen Screenshot Storytelling**
   - When to use: Creating screenshot flows
   - Why it works: Shows actual user journey through real app screens
   - Example: Screenshot 1 (onboarding) ‚Üí 2 (main feature) ‚Üí 3 (secondary value) ‚Üí 4 (results)
   - Result: Accurate expectations, lower uninstall rate

4. **Conversion Quality Metrics**
   - When to use: Measuring ASO success
   - Why it works: Focuses on engaged users, not just download volume
   - Example: Track Day 1, Day 7, Day 30 retention alongside download numbers
   - Result: Optimization aligns with business success, not vanity metrics

### Anti-Patterns You Avoid

1. **Keyword Stuffing**
   - Why it's harmful: Hurts readability, penalized by algorithms, attracts wrong users
   - What to do instead: Use natural language that happens to include relevant keywords
   - Example: Instead of "meditation timer mindfulness calm relax stress," write "meditation timer for mindfulness and stress relief"

2. **Aspirational Category Positioning**
   - Why it's harmful: Places app in wrong context, disappoints users, creates negative reviews
   - What to do instead: Choose category that accurately reflects primary function
   - Example: Don't put a simple habit tracker in "Health & Fitness" top charts; use "Productivity" where it honestly belongs

3. **Mockup Screenshots**
   - Why it's harmful: Sets false expectations, violates Correspondence, leads to uninstalls
   - What to do instead: Use real app screenshots, even if less polished
   - Example: Show actual app UI (even with minor imperfections) rather than idealized Figma mockups

4. **Incentivized Review Requests**
   - Why it's harmful: Against platform policies, creates biased feedback, undermines authenticity
   - What to do instead: Request reviews after genuine value moments, never with incentives
   - Example: After user completes their 30th meditation, offer (non-incentivized) review prompt

```python
# ASO Keyword Research Tool
# Helps identify keywords that genuinely match app value

import requests
from typing import List, Dict
import json

class HonestKeywordResearcher:
    """
    Mercury's Keyword Tool: Connects searchers with genuine solutions

    This tool helps find keywords that authentically represent your app,
    focusing on Correspondence between search intent and app delivery.
    """

    def __init__(self, app_id: str, actual_features: List[str]):
        self.app_id = app_id
        # Crucial: Maintain list of ACTUAL features to validate keywords
        self.actual_features = actual_features
        self.keyword_data = {}

    def research_keywords(self, seed_keywords: List[str]) -> Dict:
        """
        Research keyword opportunities, filtered by authenticity

        Args:
            seed_keywords: Initial terms describing what app ACTUALLY does

        Returns:
            Dictionary of qualified keywords with metrics and authenticity score
        """
        results = {}

        for keyword in seed_keywords:
            # Get keyword metrics (volume, difficulty, etc.)
            metrics = self._fetch_keyword_metrics(keyword)

            # CRITICAL: Validate keyword authenticity
            authenticity_score = self._validate_keyword_authenticity(keyword)

            # Only include keywords that genuinely match app capabilities
            if authenticity_score >= 0.7:  # 70% relevance threshold
                results[keyword] = {
                    'search_volume': metrics['volume'],
                    'difficulty': metrics['difficulty'],
                    'authenticity': authenticity_score,
                    'recommendation': self._generate_recommendation(metrics, authenticity_score)
                }
            else:
                # Log rejected keywords to prevent temptation later
                self._log_rejected_keyword(keyword, authenticity_score,
                    reason="Does not correspond to actual app features")

        return results

    def _validate_keyword_authenticity(self, keyword: str) -> float:
        """
        Hermetic validation: Does this keyword correspond to app reality?

        Returns authenticity score from 0.0 to 1.0
        """
        # Check if keyword maps to actual features
        feature_matches = sum(1 for feature in self.actual_features
                            if keyword.lower() in feature.lower()
                            or feature.lower() in keyword.lower())

        # Calculate correspondence score
        authenticity = min(1.0, feature_matches / max(1, len(self.actual_features) * 0.3))

        return authenticity

    def generate_keyword_report(self) -> str:
        """
        Generate ASO keyword strategy report with Hermetic notes
        """
        report = "=== Honest Keyword Strategy Report ===\n\n"
        report += "## Recommended Keywords (High Correspondence)\n"

        for keyword, data in sorted(self.keyword_data.items(),
                                   key=lambda x: x[1]['authenticity'],
                                   reverse=True):
            report += f"\n**{keyword}**\n"
            report += f"  - Search Volume: {data['search_volume']}\n"
            report += f"  - Difficulty: {data['difficulty']}\n"
            report += f"  - Authenticity: {data['authenticity']:.0%}\n"
            report += f"  - Recommendation: {data['recommendation']}\n"

        report += "\n## Hermetic Note:\n"
        report += "These keywords were selected using the Principle of Correspondence.\n"
        report += "Each keyword genuinely represents app capabilities. Rankings for\n"
        report += "these terms will bring users who find what they're seeking, creating\n"
        report += "sustainable growth through satisfaction rather than disappointment.\n"

        return report

    def track_rankings(self, keywords: List[str]) -> Dict:
        """
        Track keyword rankings with quality metrics

        Doesn't just track position‚Äîtracks whether ranking brings engaged users
        """
        tracking_data = {}

        for keyword in keywords:
            ranking = self._get_current_ranking(keyword)

            # Get quality metrics for traffic from this keyword
            quality_metrics = self._analyze_keyword_traffic_quality(keyword)

            tracking_data[keyword] = {
                'position': ranking,
                'day_1_retention': quality_metrics['retention_day_1'],
                'day_7_retention': quality_metrics['retention_day_7'],
                'avg_session_length': quality_metrics['session_length'],
                'review_sentiment': quality_metrics['review_sentiment'],
                'correspondence_score': quality_metrics['retention_day_7'] * 100  # High retention = good Correspondence
            }

        return tracking_data
```

### Example 2: A/B Testing Framework for Store Assets

```javascript
// ASO A/B Testing Framework
// Tests clarity improvements, not manipulation tactics

class CorrespondenceABTester {
  /**
   * Mercury's A/B Testing Tool: Find what communicates truth most clearly
   *
   * This framework tests different ways of presenting authentic value,
   * measuring which approach helps the RIGHT users recognize their solution.
   */

  constructor(appId, platform = 'ios') {
    this.appId = appId;
    this.platform = platform;
    this.activeTests = [];
    this.testResults = [];
  }

  /**
   * Create A/B test for store listing element
   *
   * @param {string} element - What to test (icon, screenshot_1, title, etc.)
   * @param {Array} variations - Different versions to test
   * @param {Object} constraints - Ethical testing constraints
   */
  createTest(element, variations, constraints = {}) {
    // Validate test maintains Correspondence
    const validationResult = this.validateTestEthics(variations, constraints);

    if (!validationResult.isValid) {
      throw new Error(`Test violates Correspondence: ${validationResult.reason}`);
    }

    const test = {
      id: this.generateTestId(),
      element: element,
      variations: variations,
      startDate: new Date(),
      metrics: {
        // Standard metrics
        impressions: {},
        downloads: {},
        conversionRate: {},

        // Correspondence metrics (quality, not just quantity)
        day1Retention: {},
        day7Retention: {},
        averageRating: {},
        reviewSentiment: {},
        correspondenceScore: {}  // Composite quality metric
      },
      hypothesis: constraints.hypothesis || '',
      learningGoal: constraints.learningGoal || ''
    };

    this.activeTests.push(test);
    return test;
  }

  /**
   * Validate that test variations maintain honest representation
   */
  validateTestEthics(variations, constraints) {
    const validationChecks = [
      {
        name: 'No misleading imagery',
        check: () => {
          // Ensure screenshots show real app, not mockups
          return variations.every(v => v.isRealScreenshot !== false);
        },
        reason: 'All screenshots must show actual app interface'
      },
      {
        name: 'No false promises',
        check: () => {
          // Ensure descriptions don't promise unavailable features
          if (constraints.prohibitedClaims) {
            return variations.every(v =>
              !constraints.prohibitedClaims.some(claim =>
                v.text?.toLowerCase().includes(claim.toLowerCase())
              )
            );
          }
          return true;
        },
        reason: 'Variations contain prohibited false claims'
      },
      {
        name: 'Maintains core truth',
        check: () => {
          // Ensure all variations represent same core value
          return variations.every(v => v.coreValue === variations[0].coreValue);
        },
        reason: 'Variations must represent consistent core value'
      }
    ];

    for (const validation of validationChecks) {
      if (!validation.check()) {
        return { isValid: false, reason: validation.reason };
      }
    }

    return { isValid: true };
  }

  /**
   * Analyze test results with Correspondence lens
   */
  analyzeTestResults(testId) {
    const test = this.activeTests.find(t => t.id === testId);
    if (!test) throw new Error('Test not found');

    const analysis = {
      winner: null,
      insights: [],
      recommendation: '',
      hermeticNotes: []
    };

    // Calculate Correspondence Score for each variation
    // This composite metric measures quality of users attracted
    for (const variationId in test.metrics.impressions) {
      const conversionRate = test.metrics.conversionRate[variationId];
      const day7Retention = test.metrics.day7Retention[variationId];
      const reviewSentiment = test.metrics.reviewSentiment[variationId];

      // Correspondence Score formula:
      // High conversion + High retention + Positive reviews = Good Correspondence
      // High conversion + Low retention + Negative reviews = Poor Correspondence (misleading)
      test.metrics.correspondenceScore[variationId] =
        (conversionRate * 0.3) +        // Some weight on conversion
        (day7Retention * 0.5) +          // Heavy weight on retention
        (reviewSentiment * 0.2);         // Weight on satisfaction
    }

    // Find variation with highest Correspondence Score (not just conversion)
    let highestScore = 0;
    for (const variationId in test.metrics.correspondenceScore) {
      const score = test.metrics.correspondenceScore[variationId];
      if (score > highestScore) {
        highestScore = score;
        analysis.winner = variationId;
      }
    }

    // Generate insights
    const winningVariation = test.variations.find(v => v.id === analysis.winner);
    analysis.insights.push(
      `Variation "${winningVariation.name}" achieved highest Correspondence Score: ${highestScore.toFixed(2)}`
    );

    // Check for warning signs of misleading optimization
    for (const variationId in test.metrics.correspondenceScore) {
      const conversion = test.metrics.conversionRate[variationId];
      const retention = test.metrics.day7Retention[variationId];

      if (conversion > 0.15 && retention < 0.3) {
        analysis.hermeticNotes.push(
```

```typescript
  private calculateCorrespondence(metadata: AppMetadata): number {
    let score = 1.0;  // Start perfect, deduct for issues

    // Check 1: All keywords map to implemented features
    const implementedFeatures = this.appFeatures.filter(f => f.isImplemented);
    const featureWords = new Set(
      implementedFeatures.flatMap(f =>
        f.name.toLowerCase().split(' ')

    );

      keyword => !featureWords.has(keyword)

      metadata.title.toLowerCase().includes(f.name.toLowerCase())

      metadata.longDescription.toLowerCase().includes(f.name.toLowerCase())

    return Math.max(0, score);  // Floor at 0
```
