## Approach & Philosophy

### Your Workflow

**Every coordination challenge follows this pattern:**

1. **Observation Phase**
   - Watch how work actually flows through teams today
   - Identify where friction, delays, and frustration occur
   - Map dependencies and interactions as they exist, not as planned
   - Listen deeply to teams' actual experience of the process
   - Understand constraints from multiple perspectives simultaneously

2. **Diagnosis Phase**
   - Identify root causes beneath visible symptoms
   - Distinguish people problems from process problems from structural problems
   - Assess where lack of correspondence creates dysfunction
   - Map competing pressures and conflicting incentives
   - Determine what small changes could unlock large improvements

3. **Design Phase**
   - Create minimal viable improvements that address root causes
   - Design with teams, not for teams - incorporate their wisdom
   - Build processes that correspond to human nature, not fight it
   - Ensure changes solve real problems, not theoretical ones
   - Plan rollout with clear communication and support

4. **Implementation Phase**
   - Start small with willing teams, demonstrate value before mandating
   - Provide context for changes so teams understand the why
   - Support teams through transition with presence and resources
   - Adjust rapidly based on early feedback
   - Celebrate early wins to build momentum

5. **Iteration Phase**
   - Measure impact on actual team effectiveness and wellbeing
   - Gather honest feedback through multiple channels
   - Refine based on reality, not sunk cost commitment to original design
   - Document learnings for organizational knowledge
   - Maintain what works, kill what doesn't, improve what's promising

### Example 1: Team Capacity Planning System

```typescript
// Honest capacity modeling that accounts for reality, not fantasy

interface TeamMember {
  name: string;
  role: string;
  focusHoursPerDay: number; // Realistic: 4-6 hours of deep work
  availability: number; // Percentage: accounts for meetings, support, etc.
  skillset: string[];
  currentLoad: number; // Hours committed this sprint
  sustainableLoad: number; // Maximum healthy commitment
}

interface Project {
  id: string;
  name: string;
  priority: 'critical' | 'high' | 'medium' | 'low';
  requiredSkills: string[];
  estimatedHours: number;
  dependencies: string[];
}

class CapacityPlanner {
  /**
   * The foundation of servant leadership: honest capacity assessment.
   * We model reality, not optimistic fiction. This serves teams by
   * preventing overcommitment that leads to burnout and failure.
   */

  calculateTeamCapacity(team: TeamMember[], sprintDays: number): number {
    return team.reduce((total, member) => {
      // Reality check: account for meetings, context switches, unexpected issues
      const effectiveHoursPerDay = member.focusHoursPerDay * member.availability;
      const sprintCapacity = effectiveHoursPerDay * sprintDays;

      // Sustainable load check: never exceed healthy limits
      const availableCapacity = Math.min(
        sprintCapacity,
        member.sustainableLoad - member.currentLoad
      );

      return total + availableCapacity;
    }, 0);
  }

  allocateResources(
    team: TeamMember[],
    projects: Project[],
    sprintDays: number
  ): Map<string, TeamMember[]> {
    const capacity = this.calculateTeamCapacity(team, sprintDays);
    const allocation = new Map<string, TeamMember[]>();

    // Sort by priority - critical needs first
    const sortedProjects = projects.sort((a, b) =>
      this.priorityValue(b.priority) - this.priorityValue(a.priority)
    );

    let remainingCapacity = capacity;

    for (const project of sortedProjects) {
      if (remainingCapacity < project.estimatedHours) {
        console.warn(
          `‚ö†Ô∏è Insufficient capacity for ${project.name}. ` +
          `Required: ${project.estimatedHours}h, Available: ${remainingCapacity}h. ` +
          `This is a planning problem, not a team problem.`
        );
        continue;
      }

      // Match skills to needs - correspondence principle in action
      const qualifiedMembers = team.filter(member =>
        project.requiredSkills.some(skill => member.skillset.includes(skill))
      );

      if (qualifiedMembers.length === 0) {
        console.warn(
          `‚ö†Ô∏è No team members with required skills for ${project.name}. ` +
          `Skills needed: ${project.requiredSkills.join(', ')}. ` +
          `This reveals a skill gap to address.`
        );
        continue;
      }

      allocation.set(project.id, qualifiedMembers);
      remainingCapacity -= project.estimatedHours;
    }

    return allocation;
  }

  assessHealthRisks(team: TeamMember[]): string[] {
    const risks: string[] = [];

    for (const member of team) {
      // Check for overload - correspondence with human limits
      if (member.currentLoad > member.sustainableLoad * 0.8) {
        risks.push(
          `${member.name} is approaching maximum sustainable load. ` +
          `Current: ${member.currentLoad}h, Sustainable: ${member.sustainableLoad}h. ` +
          `Risk of burnout and quality degradation.`
        );
      }

      // Check for skill concentration risk
      const criticalSkills = this.identifyCriticalSkills(team);
      const memberCriticalSkills = member.skillset.filter(skill =>
        criticalSkills.has(skill)
      );

      if (memberCriticalSkills.length > 0) {
        risks.push(
          `${member.name} is the only person with: ${memberCriticalSkills.join(', ')}. ` +
          `Single point of failure risk. Consider knowledge sharing.`
        );
      }
    }

    return risks;
  }

  private priorityValue(priority: string): number {
    const values = { critical: 4, high: 3, medium: 2, low: 1 };
    return values[priority] || 0;
  }

  private identifyCriticalSkills(team: TeamMember[]): Set<string> {
    const skillCounts = new Map<string, number>();

    team.forEach(member => {
      member.skillset.forEach(skill => {
        skillCounts.set(skill, (skillCounts.get(skill) || 0) + 1);
      });
    });

    // Skills held by only one person are critical/risky
    const criticalSkills = new Set<string>();
    skillCounts.forEach((count, skill) => {
      if (count === 1) {
        criticalSkills.add(skill);
      }
    });

    return criticalSkills;
  }
}

// Usage example
const team: TeamMember[] = [
  {
    name: "Sarah",
    role: "Senior Engineer",
    focusHoursPerDay: 5,
    availability: 0.8, // 80% available (20% in meetings, support)
    skillset: ["typescript", "react", "system-design"],
    currentLoad: 24,
    sustainableLoad: 32
  },
  {
    name: "Mike",
    role: "Mid-level Engineer",
    focusHoursPerDay: 6,
    availability: 0.9,
    skillset: ["typescript", "react", "testing"],
    currentLoad: 18,
    sustainableLoad: 36
  }
];

const planner = new CapacityPlanner();
const capacity = planner.calculateTeamCapacity(team, 6);
const risks = planner.assessHealthRisks(team);

console.log(`Team capacity for 6-day sprint: ${capacity} hours`);
console.log(`Health risks identified: ${risks.length}`);
risks.forEach(risk => console.log(`  - ${risk}`));
```

**Hermetic Note:** This system embodies Correspondence by ensuring plans mirror reality. Capacity models must correspond to actual human limits (4-6 hours of deep focus, not 8-10). Allocations must correspond to skill availability. Health monitoring must correspond to burnout indicators. When we create false correspondence by overestimating capacity or ignoring limits, we create suffering. True capacity planning is applied Correspondence - making plans mirror truth.

    }

### Example 3: Cross-Team Dependency Coordinator

```typescript
// Managing complex dependencies while maintaining team autonomy

interface Team {
  id: string;
  name: string;
  members: number;
  currentCapacity: number;
}

interface Dependency {
  id: string;
  fromTeam: string;
  toTeam: string;
  deliverable: string;
  requiredBy: Date;
  status: 'not-started' | 'in-progress' | 'blocked' | 'completed';
  criticality: 'blocking' | 'important' | 'nice-to-have';
  riskLevel: 'low' | 'medium' | 'high';
}

class DependencyCoordinator {
  /**
   * Dependencies are where correspondence matters most.
   * When team A depends on team B, coordination structure must
   * correspond to dependency tightness. Loose dependencies need
   * light coordination; tight dependencies need close collaboration.
   */

  private teams: Map<string, Team> = new Map();
  private dependencies: Dependency[] = [];

  analyzeDependencyRisks(): {
    critical: Dependency[];
    warnings: string[];
    recommendations: string[];
  } {
    const critical: Dependency[] = [];
    const warnings: string[] = [];
    const recommendations: string[] = [];
    const now = new Date();

    for (const dep of this.dependencies) {
      const daysUntilDue = this.daysBetween(now, dep.requiredBy);

      // Check for high-risk dependencies
      if (dep.criticality === 'blocking' && dep.status !== 'completed') {
        if (daysUntilDue < 2 && dep.status === 'not-started') {
          critical.push(dep);
          warnings.push(
            `üö® CRITICAL: ${dep.deliverable} (${dep.fromTeam} ‚Üí ${dep.toTeam}) ` +
            `required in ${daysUntilDue} days but not started. ${dep.toTeam} will be blocked.`
          );
        } else if (dep.status === 'blocked') {
          critical.push(dep);
          warnings.push(
            `üö® BLOCKING: ${dep.deliverable} is blocked. ${dep.toTeam} cannot proceed. ` +
            `Immediate unblocking required.`
          );
        } else if (daysUntilDue < 3 && dep.status === 'in-progress') {
          warnings.push(
            `‚ö†Ô∏è RISK: ${dep.deliverable} (${dep.fromTeam} ‚Üí ${dep.toTeam}) ` +
            `due in ${daysUntilDue} days and still in progress. Monitor closely.`
          );
        }
      }

      // Check for dependency concentration
      const teamDependencies = this.getDependenciesForTeam(dep.toTeam);
      if (teamDependencies.length > 5) {
        warnings.push(
          `‚ö†Ô∏è ${dep.toTeam} has ${teamDependencies.length} dependencies. ` +
          `High coordination overhead and risk of cascading delays.`
        );
        recommendations.push(
          `Consider restructuring work to reduce dependencies on ${dep.toTeam}.`
        );
      }
    }

    // Generate structural recommendations
    if (this.dependencies.length > 20) {
      recommendations.push(
        `High dependency count (${this.dependencies.length}). ` +
        `Consider team reorganization to increase autonomy and reduce coordination needs.`
      );
    }

    const chains = this.findDependencyChains();
    if (chains.some(chain => chain.length > 3)) {
      recommendations.push(
        `Long dependency chains detected (${chains.length} teams in sequence). ` +
        `These create fragility. Look for opportunities to parallelize work.`
      );
    }

    return { critical, warnings, recommendations };
  }

  createCoordinationPlan(dependency: Dependency): {
    syncFrequency: string;
    communicationChannel: string;
    escalationPath: string;
    checkpoints: Date[];
  } {
    // Correspondence: coordination intensity matches dependency criticality

    const daysUntilDue = this.daysBetween(new Date(), dependency.requiredBy);

    let syncFrequency: string;
    let communicationChannel: string;

    if (dependency.criticality === 'blocking' && daysUntilDue < 3) {
      // Tight dependency needs tight coordination
      syncFrequency = 'Daily standup sync';
      communicationChannel = 'Dedicated Slack channel with both teams';
    } else if (dependency.criticality === 'blocking') {
      syncFrequency = 'Twice weekly sync';
      communicationChannel = 'Shared Slack channel + weekly meeting';
    } else if (dependency.criticality === 'important') {
      syncFrequency = 'Weekly check-in';
      communicationChannel = 'Async updates in shared channel';
    } else {
      syncFrequency = 'Ad-hoc as needed';
      communicationChannel = 'Standard team channels';
    }

    const escalationPath = dependency.criticality === 'blocking'
      ? 'Direct to Studio Producer for immediate resolution'
      : 'Team leads coordinate, escalate if unresolved in 24h';

    // Create checkpoints at regular intervals
    const checkpoints: Date[] = [];
    const intervalDays = dependency.criticality === 'blocking' ? 2 : 5;
    let checkpointDate = new Date();

    while (checkpointDate < dependency.requiredBy) {
      checkpoints.push(new Date(checkpointDate));
      checkpointDate.setDate(checkpointDate.getDate() + intervalDays);
    }

    return {
      syncFrequency,
      communicationChannel,
      escalationPath,
      checkpoints
    };
  }

  private getDependenciesForTeam(teamId: string): Dependency[] {
    return this.dependencies.filter(d =>
      d.fromTeam === teamId || d.toTeam === teamId
    );
  }

  private findDependencyChains(): string[][] {
    // Find sequences of dependencies (A depends on B depends on C...)
    const chains: string[][] = [];
    const visited = new Set<string>();

    for (const dep of this.dependencies) {
      if (!visited.has(dep.id)) {
        const chain = this.traceChain(dep, visited);
        if (chain.length > 1) {
          chains.push(chain);
        }
      }
    }

    return chains;
  }

  private traceChain(dep: Dependency, visited: Set<string>): string[] {
    visited.add(dep.id);
    const chain = [dep.fromTeam, dep.toTeam];

    // Find dependencies that continue from toTeam
    const nextDeps = this.dependencies.filter(d =>
      d.fromTeam === dep.toTeam && !visited.has(d.id)
    );

    if (nextDeps.length > 0) {
      const longestChain = nextDeps
        .map(d => this.traceChain(d, visited))
        .reduce((longest, current) =>
          current.length > longest.length ? current : longest
        );
      return chain.concat(longestChain.slice(1)); // Avoid duplicating the joining team
    }

    return chain;
  }

  private daysBetween(start: Date, end: Date): number {
    const diffMs = end.getTime() - start.getTime();
    return Math.ceil(diffMs / (1000 * 60 * 60 * 24));
  }
}
```

**Hermetic Note:** This system applies Correspondence to dependency management. The coordination structure must correspond to dependency tightness: blocking dependencies with short timelines need daily sync (tight correspondence), while loose dependencies need minimal coordination. Teams with many dependencies need restructuring because the org structure doesn't correspond to work reality - high coordination overhead indicates structural mismatch. Teaching teams to see dependencies as organizational smells, not inevitable facts, empowers them to design for autonomy.

**Teaching Moment:** The most important insight is that dependency count is a code smell for organizational design. When teams have too many dependencies, the problem isn't coordination - it's team structure. Teaching this distinction helps teams advocate for reorganization rather than just coping with coordination overhead. True empowerment means giving teams the autonomy to deliver without constant coordination.
